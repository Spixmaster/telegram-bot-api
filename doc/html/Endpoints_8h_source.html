65  * By default, updates starting with the earliest unconfirmed update are returned. An update is considered confirmed as soon as getUpdates is called with
66  * an offset higher than its update_id. The negative offset can be specified to retrieve updates starting from -offset update from the end

68  * @param limit: Limits the number of updates to be retrieved. Values between 1—100 are accepted. Defaults to 100.
69  * @param timeout: Timeout in seconds for long polling. Defaults to 0, i.e. usual short polling. Should be positive,
70  * short polling should be used for testing purposes only.
71  * @param allowed_updates: List the types of updates you want your bot to receive. For example, specify [“message”, “edited_channel_post”, “callback_query”] to only
72  * receive updates of these types. See Update for a complete list of available update types. Specify an empty list to receive all updates regardless
73  * of type (default). If not specified, the previous setting will be used. Please note that this parameter doesn't affect updates created
74  * before the call to the getUpdates, so unwanted updates may be received for a short period of time.
75  * @return Update object
76  */
